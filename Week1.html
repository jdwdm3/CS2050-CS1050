SSH into our Server

Like mentioned above, we are going to be connecting to a server that is here on campus through what is called SSH (Secure Shell). Secure Shell provides strong authentication and secure encrypted data communications between two computers connecting over an insecure network such as the Internet.

Mac -- Terminal

In order to utilize SSH on a Mac, we are going to ironically use it's command line that is built in to a program called Terminal. Another cool thing if you didn't know this already, is a Mac is based off of a UNIX operating system. There are EXTREME similarties between UNIX and LINUX, so majority of the commands we will be using on our Linux server, you will be able to use in Terminal as well. Linux Commands (Basics)

There are a TON of commands that can be used at this point in order to navigate yourself around, and also perform various functionality. With this being an introduction course, we are going to focus on a couple of core navigation commands. The commands I expect you all to know very well by the end of Lab/Week1 are the following:

  • ls (list items in directoy)
  • mkdir *NameOfDir* (make a new directory)
  • cd *NameOfDir* (change directory)
  • vim *filename* (open a file in the vim text editor)
    • :wq (write Quit, or save)
    • :q! (quit without saving)
    • :*LineNumber* (Moves Curser to that line)
  • gcc -Wall -Werror (compile a .c file into an executable file)
  • cp *Original_File* *OtherFile* (copies a file into another [A way to change name of file on command line])
  • mv *File_Name* *New_Location* (moves a file from one location, to another)
  • clear (clears command screen of all previous outputs)
  • rm *File_Name* (removes a file)
